How it works

Four steps from transaction to decision

WRAITH processes every transaction through a real-time ML pipeline — no human in the loop, no delay.

01
Transaction arrives
Your payment API sends a transaction to WRAITH via webhook or REST. Amount, device, location, account — captured instantly.
≤ 2ms ingestion
02
16 features computed
Velocity, z-score, geo-risk, device novelty, structuring probe, time-of-day, channel risk — all enriched from account history in real time.
real-time enrichment
03
5 models score in parallel
Isolation Forest, Gradient Boost, LSTM, Autoencoder, and Rule Engine all vote simultaneously. Weighted ensemble produces a 0–100 risk score.
ensemble ml
04
Decision + action
Low → pass. Medium → log. High → review. Critical → block + alert + SAR. Your system receives the verdict in under 50ms.
≤ 50ms decision
